# Sharp Odds × Retail Odds × Polymarket: 三源价值下注框架

## 核心公式

```
Value = P(Pinnacle真实概率) − P(Polymarket价格) − 摩擦力
```

**当 Value > 0 → 正向期望（+EV）信号。**

## 第一步：获取 Sharp 基准概率（去抽水）

Pinnacle 是公认最 Sharp 的庄家。先去抽水得到真实概率：

### Multiplicative 去抽水法

```
P真实(热刺) = (1/赔率热刺) / (1/赔率热刺 + 1/赔率平局 + 1/赔率埃弗顿)
```

**案例——热刺 vs 埃弗顿（2026-05-24）Pinnacle 赔率：**

| 结果 | 赔率 | 隐含概率 | 去抽水后（真实概率） |
|:---|:---:|:---:|:---:|
| 热刺胜 | 1.97 | 50.76% | **49.3%** |
| 平局 | 3.57 | 28.01% | **27.2%** |
| 埃弗顿胜 | 4.14 | 24.15% | **23.5%** |
| **合计** | | **102.92%**（抽水2.92%） | **100%** |

Sharp 真实概率 ≈ 市场最接近"真实"的估计。

## 第二步：对比 Polymarket 价格

Polymarket 价格直接是概率（0.505 = 50.5%），无需转换。

| 结果 | Pinnacle真实概率 | Polymarket价格 | 差距 | 方向 |
|:---|:---:|:---:|:---:|:---|
| 热刺胜 | 49.3% | 50.5% | +1.2% | 热刺 Yes 轻度高估 |
| 平局 | 27.2% | 26.5% | -0.7% | 平局 Yes 轻度低估 ✅ |
| 埃弗顿胜 | 23.5% | 23.5% | 0.0% | 精确匹配 |

**信号强度分级**：

| 差距 | 强度 | 行动 |
|:---:|:---:|:---|
| < 2% | ↔️ 中性 | 市场已定价，无价值 |
| 2-5% | 🟢 轻度 | 可轻仓限价单 |
| 5-10% | 🟡 明显 | 可入场，注意摩擦力 |
| > 10% | 🔴 强烈 | 可能有信息差，但检查流动性 |

## 第三步：检查订单薄深度（摩擦力的决定因素）

Polymarket 上即使价格"正确"，如果深度不够，吃单滑点会吃掉利润。

**使用 CLOB API 检查：**
```
GET /book?token_id=YES_TOKEN_ID
```

**关注的指标**：
- **价差（best bid ↔ best ask）**：越小越好（< 0.01 很好）
- **前3档总深度**：至少 > $5,000 才值得入场
- **挂Maker单（限价）**：手续费为0，还有25%返佣；吃单手续费3%

## 第四步：附加信号——一周资金流向

Polymarket 的 `oneWeekPriceChange` 字段显示过去7天价格变化：

- **正变化** → 资金流入该方向
- **负变化** → 资金流出

**使用场景**：
1. **追趋势**：如果一周价格 +5% 且阵容利好落地，可能趋势已到头
2. **反向押注**：如果一周被砸了 7.5% 但基本面没有实质变化 → 可能是过度反应
3. **与Sharp对比**：如果趋势方向和Sharp真实概率方向一致 → 信号更强

## 第五步：用实际首发阵容校准

赔率是基于"预期阵容"形成的。实际首发出来后需要重新评估：

| 信号 | 对价值的影响 |
|:---|:---|
| 你方核心球员 X 缺阵 | 你方概率应下调 → 原有+EV信号可能消失 |
| 对方核心球员 X 缺阵 | 对方概率应下调 → 原有你方+EV信号可能增强 |
| 双方均完整 | 维持原分析 |

**案例分析（热刺 vs 埃弗顿）**：
- 赛前Pinnacle给热刺49.3%
- 实际首发：热刺（范德芬+帕利尼亚回归，强于预期）+ 埃弗顿（4核心缺阵，弱于预期）
- → 热刺真实概率应高于49.3% → Polymarket的0.505从"轻度高估"变为"合理/轻度低估"
- → 调整方向：🟢 轻度看多热刺 Yes

## 完整工作流

```
① 从 Pinnacle 获取赔率 → 去抽水 → 得 Sharp 真实概率
② 从 Polymarket Gamma API 获取对应市场事件价格
③ 计算差距 Value = P(Sharp) − P(Polymarket)
④ 检查订单薄深度（价差、前3档深度）
⑤ 查看一周资金流向（oneWeekPriceChange）
⑥ 实际首发出来后重新校准
⑦ 如果有 3-5%+ 的差距 + 足够的深度 → 用Maker限价单入场
   如果 3源共振（Sharp < Retail < Polymarket 或反之）→ 加重仓位
```

## 常见陷阱

| 陷阱 | 后果 | 对策 |
|:---|:---|:---|
| Pinnacle开盘 vs 闭盘 | 开盘1.85收盘1.97，差6% | **仅用闭盘前2小时的赔率** |
| Polymarket小盘口深度浅 | 价差宽+滑点大 | 先用CLOB API检查深度 |
| 手续费吃掉微利 | 3%吃单费=信号4.5%只剩1.5% | **只用Maker限价单** |
| 首发射击（阵容出炉后赔率突变） | 赛前赔率已失效 | 等首发确认后再做终端交易 |
