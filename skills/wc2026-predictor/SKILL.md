# WC2026 世界杯比赛预测系统

## 概述

基于已积累的球队情报（数据根目录：`~/wc2026/`），赛前收集赔率/天气/新闻，10维度评分生成预测。**预测前必须从 FotMob 验证赛程**（模板生成的 match pages 曾被证实对阵/日期全错）。

### 知识库结构

所有数据在 `~/wc2026/` 下：
```
~/wc2026/
├── entities/         ← 48支球队实体页（阵容/伤病/热身赛数据）
├── matches/小组赛/   ← 74场小组赛独立页面（战术/赔率/天气/复盘）
├── comparisons/      ← 73组历史交锋数据
├── warmup/results/   ← 42场热身赛结果
├── wiki/concepts/    ← 裁判/伤病追踪/场地分析
└── raw/articles/     ← 153篇原始采集文章
```

## 赛前预测执行流程

有两个运行模式，由 cron 时间决定：

### 早间版（7:00）—— 情报采集模式

采集所有情报，不做最终评分。产出情报简报。

```
① browser_navigate→FotMob 获取真实赛程 fotmob.com/leagues/77/fixtures/world-cup
② 查 ~/wc2026/entities/{队名}.md 获取阵容/伤病/热身赛数据
③ 搜赔率: The Odds API + Polymarket
④ 搜索最新情报 — **Parallel 免费 MCP + web_extract 降级 browser, 每场 100 篇**
   （同上 12 维度分布）
④.5 对照本地实体库，交叉验证 — 更新 entity 最新动态
⑤ 🌤️ 露天场馆天气 → Open-Meteo API
⑥ 输出情报简报 → 告知用户"📋 情报已更新，晚间 20:00 出最终预测"
```

### 晚间版（20:00）—— 最终预测模式

在早间情报基础上，做最终评分预测。

```
① 快速验证：FotMob 确认赛程是否有变
② 确认关键信息：首发阵容确认？突发伤病？赔率大幅变动？
③ 10维度评分 + 特殊情景修正 + 净需求分析
④ 输出预测报告 → 写入临时文件并 lark-cli --markdown 推送，回复简短确认
```

**🔴 交互模式**：确认采集范围，指定要预测的比赛。cron 模式自动跳过。

## 10维度评分框架

```
 1. 实力评估（FIFA排名 + 球员身价 + 大赛经验）
 2. 近期状态（近6场战绩 + 进球/失球趋势）
 3. 阵容完整性（伤病影响 + 核心球员可用性）
 4. 战术对位（阵型相克 + 风格克制）
 5. 历史交锋（心理优势 + 比分模式）
 6. 🔥 外部因素（主客场 + 天气/温度/湿度 + 海拔 + 休息天数）
 7. 关键球员因素（球星个人能力 + 对位优势）
 8. 大赛心理（历史表现 + 淘汰赛经验）
 9. 名宿观点参考（多位名宿预测，按历史准确率加权）
10. 深层数据参考（xG/控球率/射门转化率等）
```

## 特殊情景修正

| 情景 | 影响 | 修正 |
|:-----|:-----|:----:|
| 🔥 复仇战 | 上届被淘汰对手 | +0.3 |
| 🌙 早场/夜场 | 不适应当地时间 | -0.3 |
| 🔄 已出线轮换 | 可能上替补 | -0.5~1.0 |
| 👋 老将告别战 | 最后一届 | +0.3 |
| ⚡ 连续作战 | 休息不足3天 | -0.5 |
| 🏠 接近主场 | 地理优势 | +0.5 |
| ✈️ 长途飞行 | 跨大陆后比赛 | -0.3 |
| 🗣️ 赛前狂妄言论反指标 | 队长/教练公开说"我们>对手"/"要统治比赛"，对手获得额外动力 | -0.2（发言方）, +0.2（对手方） |
| 👨‍💼 教练大胆变阵 | 首发阵容2+个意外调整（弃用常规主力/启用新人），结果波动性上升 | ±0.3（不确定性上升） |
| 🏴󠁧󠁢󠁳󠁣󠁴󠁿 长期缺席重返焦虑 | 世界杯缺席20年+的球队首战效率下降，数据(控球率/xG)≠比分 | -0.3~0.5（进攻端效率） |
| 🅿️ 低控球率高效率反直觉模式 | 惯用5-4-1/5-3-2且射门转化率>15%的球队vs控球型球队，市场赔率系统性低估胜率 | +0.3（防守方胜率修正） |
| 🏜️ 极端高温(露天, 30-35°C) | 冷适应方 vs 热适应方 | -0.5（冷适应方） |
| 🏜️ 极端高温(露天, 35°C+) | 冷适应方 vs 热适应方 | -0.7~1.0（冷适应方） |
| 🏜️ 极端高温(露天) | 双方都不适应 | -0.3（双方） |
| 🏔️ 高海拔(2200m墨西哥城) | 低海拔队首次 | -0.5~0.8 |
| 🏔️ 中高海拔(1560m瓜达拉哈拉) | 低海拔队 | -0.3~0.5 |
| 🌧️ 暴雨/雷阵雨 | 草皮湿滑影响技术型 | -0.2~0.3 |
| 📉 只需1分即出线/保级 | 75分钟后收缩死守,不反扑 | -1.0~1.5（进攻端） |
| 🎭 揭幕战情绪因子 | 大赛首场红牌/情绪失控概率高 | -0.3（双方） |
| 💪 大赛首战斗志加成 | 中下游球队在赛事首战展现拼搏斗志和超常体能 | +0.3（中下游球队） |
| 🪄 替补建功效应 | 大赛首轮替补球员出场后进球概率升高（韩国5名替补世界杯首秀进球） | +0.2（替补端威胁评估） |
| 🏠 东道主首战紧张因子 | 东道主首场比赛上半场效率显著偏低，70分钟后才进入状态（加美墨三东道主一致） | -0.2（上半场进攻端效率） |
| ⭐ 保护性轮换模式 | 大比分领先时教练提前保护核心球员（Pulisic半场被换下） | -0.3（核心球员预期出场时间） |

## 🔑 净需求 > 纸面实力

| 情景 | 正确推理 |
|:-----|:---------|
| A队只需1分出线 | 70分钟后收缩死守，不会冒险 → 平局概率↑ |
| A队必须赢、B队已出线轮换 | A队拼命 → 胜率↑ |
| A队已出线、B队必须赢 | A队轮换替补 → B队胜率↑↑ |

## 赔率信号

| 来源 | 特性 |
|:-----|:-----|
| Polymarket | 信息效率高，对新闻反应快 |
| The Odds API (Sharp均值) | 资金深度大，更稳定 |
| ⭐ 两者价差 > 5% | 分歧信号，最有价值的预测输入 |

## 运行方式

本技能由 Hermes 定时任务系统调度执行（`cronjob action='list'` 可查当前配置）。建议赛前 4-6h 运行预测，赛后及时复盘。
